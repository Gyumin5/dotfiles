# /handoff: 세션 인수인계 문서 생성

현재 세션의 작업 내용을 다음 세션(또는 다른 에이전트)에 넘길 수 있도록 정리한다.

## 워크플로우

### 1. 현재 세션 분석

대화 내용을 분석하여 아래 항목을 추출:

- 작업 목표 (무엇을 하려 했는가)
- 완료된 작업 (무엇이 끝났는가)
- 진행 중인 작업 (무엇이 진행 중인가)
- 미해결 이슈 (블로커, 결정 필요 사항)
- 수정된 파일 목록

### 2. handoff.md 생성

프로젝트 루트에 `handoff.md`를 생성:

```markdown
# Handoff: [작업 제목]

## 목표

(전체 목표)

## 완료

- [x] ...
- [x] ...

## 진행 중

- [ ] ... (현재 상태: ...)
- [ ] ...

## 미해결

- ⚠️ ... (결정 필요)
- 🐛 ... (버그)

## 수정된 파일

- `path/to/file.ts` - 변경 내용 요약
- `path/to/other.py` - 변경 내용 요약

## 다음 세션 시작 시

1. `handoff.md`를 읽어라
2. (구체적 다음 단계)
3. 완료 후 `handoff.md` 삭제
```

### 3. git 상태 포함

`git diff --stat`과 `git status`를 포함하여 커밋되지 않은 변경사항도 기록.

## 규칙

- 간결하게 작성 (100줄 이내)
- 추상적 표현 금지, 구체적 파일명과 명령어만
- 기존 handoff.md가 있으면 덮어쓸지 사용자에게 확인
- 생성 후 내용을 보여줘서 사용자가 검토할 수 있게
